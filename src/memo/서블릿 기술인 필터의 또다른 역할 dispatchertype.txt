우리는 이전 시간에 예외 객체 발생 시 혹은 response.sendError 발생 시 WAS의 설정된 정보에 따라 에러 페이지를 만드는 컨트롤러로 실행 흐름이 다시 이동하는 것을 알아 보았다. 
그런데 이떄 필터/인터셉터도 다시 실행이 된다. 
그런데 이건 매우 비효율적이다. 이미 필터/인터셉터를 한번 실행했는데 또 실행을 하는 것인 비효율이다.
예를 들어 로그인 인증 필터를 거쳤는데 다시 또 거치는 것은 비효율적이다. 
이러한 비효율을 막기 위해서 Tomcat(WAS)에는 dispacher type이라는 서블릿 스펙이 있다. (dispatcher type은 jakarata.servlet.dispatcher이기에 서블릿 기술이다)
이것은 어떤 요청이 클라이언트 요청인지 아니면 에러에 의한 재요청인지를 구분하여 필터를 재실행을 컨트롤하기 위함이다. 

dispatcher type == REQUEST
-> 클라이언트 요청

dispatehr type == ERROR
-> 에러 요청

이번 시간에는 dispatcher type에 따라 필터가 어떻게 움직이는 지 알아보자.
