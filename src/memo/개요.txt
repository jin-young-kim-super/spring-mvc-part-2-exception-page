우리는 이전시간에 서블릿/스프링이 어떻게 예외를 처리하는지 학습을 하였다. 
이번 시간에는 스프링의  API 예외까지 알아 볼 것이다.


이전 시간에는 예외 발생 시 사용자에게 오류 [화면]을 보여 주는 것이였다.
그러나 예를 들어, 앱이 서버에 통신을 하다가 예외가 터지면 이때는 오류 [화면]을 보여줄 수는 없다.
또한 MSA처럼 서버와 서버끼리 통신 중의 예외 발생 시에도 그렇다.
기업 간의 서비스 통신 시에도 그러하다.
-> 즉 이러한 떄에는 오류 발생 시, 어떠한 예외 데이터를 보낼 지 서로 약속한 정확한 데이터를 보내야 한다
오류화면을 보내는 것이 아니라! 왜냐하면 클라이언트가 만약 웹 브라우저라면 HTML 오류 화면을 보내주면 렌더링을 해서 예외 정보를 볼 수가 있지만 위와 같은
클라이언트 언트가 웹 브라우저가 아닌 경우에는 HTML 오류 페이지를 응답 받아도 이걸 해석할 능력이 없다.

이런 문제를 해결하기 위하여 오류 페이지 【컨트롤러】에서 오류 페이지뿐만 아니라 JSON 형식으로 오류 정보를 보낼 수 있게 수정을
해주면 된다. 이전과 같이 서블릿 방식으로 먼저 다뤄보고 그 다음 스프링이 제공하는 방식으로 JSON 형식 오류를 다루어 볼 것이다.
